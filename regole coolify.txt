regole per deploy su Coolify

  Stack

  - Astro (static/SSR)
  - EmailJS per il form (client-side, zero backend)

  Regole build

  - npm run build deve funzionare in locale prima di pushare
  - package-lock.json pulito e committato nel repo

  Regole Coolify

  - Build pack: Nixpacks (auto-detecta Astro)
  - Il repo deve essere collegato tramite la GitHub App configurata su Coolify
  - Nessun Dockerfile necessario
  - Nessuna variabile d'ambiente necessaria (le chiavi EmailJS sono nel
  frontend)

  ENV VAR OBBLIGATORIE (problema risolto precedentemente)

  Problema 1 — Binding nativo mancante:
    Astro con Vite 8+ usa rolldown che ha binari platform-specific.
    Il lockfile generato su macOS/Windows non include @rolldown/binding-linux-x64-gnu.
    Fix: forzare reinstallazione fresca su Linux.

  Problema 2 — Nessun comando di avvio per output statico:
    Nixpacks rileva Astro come "node" ma non sa servire i file statici buildati.
    Fix: usare `npx serve` per servire la cartella dist.

  Le tre env var da impostare SEMPRE su Coolify per questo progetto:
    NIXPACKS_INSTALL_CMD=rm -rf node_modules package-lock.json && npm install
    NIXPACKS_START_CMD=npx serve dist -l 4321 -s
    NIXPACKS_NODE_VERSION=22
