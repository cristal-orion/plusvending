regole per deploy su Coolify

  Stack

  - Astro (static output)
  - EmailJS per il form (client-side, zero backend)
  - Nginx come web server per i file statici

  Regole build

  - npm run build deve funzionare in locale prima di pushare
  - NON committare package-lock.json (il Dockerfile fa npm install fresco
    per ottenere i binari nativi corretti per Linux)

  Regole Coolify

  - Build pack: Dockerfile (NON Nixpacks)
  - Il repo deve essere collegato tramite la GitHub App configurata su Coolify
  - Il Dockerfile nella root gestisce tutto: build con Node 22 Alpine + serve con Nginx
  - Porta esposta: 4321
  - Nessuna variabile d'ambiente necessaria (le chiavi EmailJS sono nel frontend)

  Note storiche

  Nixpacks non funzionava per due motivi:
  1. Binding nativo mancante — rolldown richiede binari Linux non presenti
     nel lockfile generato su Windows/macOS
  2. Nessun start command per output statico — Nixpacks non sa servire
     i file buildati di Astro
  Soluzione definitiva: Dockerfile multi-stage (node:22-alpine + nginx:alpine)
